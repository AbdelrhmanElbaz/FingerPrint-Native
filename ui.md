# ui.md — تصميم الواجهة وتجربة الاستخدام

## 1. المبدأ العام

نافذة رئيسية واحدة (`QMainWindow`)، RTL بالكامل، بنفس الهوية البصرية الحالية (الألوان والخطوط) لكن كعناصر Qt حقيقية بدل CSS مُحقن. لا يوجد "صفحات" بمفهوم الويب — يوجد **`QStackedWidget`** مركزي يبدّل بين الشاشات، و**Sidebar** ثابت (Dock Widget) لشجرة الملفات.

### لوحة الألوان (منقولة كـ QSS بدل CSS)
| المتغيّر | القيمة | الاستخدام |
|---|---|---|
| `--bg` | `#faf9f5` | خلفية عامة |
| `--card` | `#efe9de` | كروت / KPI / Expanders |
| `--border` | `#e6dfd8` | حدود |
| `--accent` | `#cc785c` | أزرار أساسية، Hover، تركيز |
| `--success` | `#5db872` | أزرار التحميل/التأكيد |
| `--warning` | `#d4a017` | تنبيهات |
| `--danger` | `#c64545` | أخطاء/حذف |
| `--text` | `#141413` | نص أساسي |
| `--text-muted` | `#6c6a64` | تسميات/Captions |

الخط: `IBM Plex Sans Arabic` (يُضمَّن كـ Font Resource داخل الـ exe لضمان ظهوره على أي جهاز حتى لو مش مثبَّت).

## 2. هيكل النافذة الرئيسية

```
┌──────────────────────────────────────────────────────────┐
│  MenuBar (اختياري: ملف / إعدادات / حول البرنامج)          │
├───────────────┬────────────────────────────────────────────┤
│               │  Header: اسم الشركة + الشهر المفتوح حاليًا   │
│  Sidebar       ├────────────────────────────────────────────┤
│  (شجرة الملفات) │                                            │
│  🏢 شركة        │      QStackedWidget (المحتوى المتغيّر)      │
│    📅 سنة       │      - DashboardView                       │
│      📄 شهر     │      - EmployeeDetailView                  │
│  ─────────      │      - ImportView (لو مفيش ملف مفتوح)      │
│  📤 فتح مؤقت    │                                            │
│  ─────────      │                                            │
│  ⚙️ إعدادات     ├────────────────────────────────────────────┤
│               │  StatusBar: عدد التعديلات المعلَّقة + Apply/Discard │
└───────────────┴────────────────────────────────────────────┘
```

- **Sidebar** = `QTreeWidget` قابل للطي/الفتح (شركة → سنة → شهر)، مع قائمة سياقية (Right-click) لكل عنصر: فتح / حذف / إعادة تسمية — بدل الـ Popover الحالي.
- **StatusBar** = بديل الـ "Floating Apply Bar" الحالي: شريط سفلي دائم يظهر فيه "عدد التعديلات المعلَّقة" + زرّي (تطبيق / تجاهل) يُفعَّلا فقط لو فيه تعديلات معلَّقة. لا حاجة لـ overlay عائم متحرك.

## 3. الشاشات التفصيلية

### 3.1 شاشة الاستيراد (Import View)
تظهر لو مفيش ملف مفتوح حاليًا، أو عند الضغط على "استيراد ملف جديد".

- زر/منطقة سحب-وإفلات لملف `.xls/.xlsx` (`QFileDialog` + Drag&Drop مدعوم في Qt أصلًا بدون أي Hack).
- Checkbox: "فتح مؤقت (Anonymous) — بدون حفظ".
- لو مش Anonymous: 3 حقول (اختيار شركة موجودة أو "شركة جديدة" + سنة + شهر) — نفس منطق الاستيراد الحالي.
- زر "استيراد" يُفعَّل فقط بعد اختيار ملف + بيانات كاملة.
- **الجديد:** شريط تقدّم (`QProgressDialog`) أثناء التحليل — لأن ملفات البصمة الكبيرة ممكن تاخد وقت، والمستخدم محتاج يعرف إن البرنامج شغّال مش عالق (مشكلة معروفة في Streamlit مع rerun المتكرر).

### 3.2 شاشة استعادة آخر جلسة
عند فتح البرنامج ولقى آخر شهر مفتوح محفوظ: رسالة ترحيبية بسيطة "آخر ملف: [شركة] — [شهر] [سنة]" مع زرّين: "فتح آخر ملف" / "استيراد ملف جديد". (نفس السلوك الحالي، بدون تغيير جوهري).

### 3.3 الشاشة الرئيسية / لوحة التحكم (Dashboard View)
تظهر بعد فتح ملف شهر:

1. **إعدادات الرواتب** (أعلى الشاشة): جدول أسعار الساعة لكل موظف (Grid من `QLineEdit`/`QDoubleSpinBox`)، مع Checkbox "عرض الغائبين طول الشهر أيضًا"، وزر 🔍 بجانب كل موظف يفتح `EmployeeDetailView`.
2. **إحصائيات عامة (KPI Cards):** 4 كروت أفقية (عدد الموظفين / إجمالي الرواتب / إجمالي الساعات / عدد ذوي البصمات الناقصة) — `QFrame` بستايل الكارت الحالي.
3. **البحث والتصفية:** حقل بحث بالاسم/الـ ID + حد أدنى/أقصى للراتب، تطبّق فورًا على الجدول (بدون Rerun — مجرد `QSortFilterProxyModel`).
4. **جدول كشف الرواتب:** `QTableView` مبني على `QAbstractTableModel` (وليس `st.dataframe`) — يدعم فرز، تحديد، ونسخ.
5. **الرسوم البيانية:** تبويبات (`QTabWidget`) لثلاث رسومات (توزيع الرواتب / ساعات العمل / أيام الحضور) بـ `pyqtgraph` أو `QtCharts`.
6. **أزرار التحميل:** كشف الرواتب فقط / الملف النهائي الكامل — تفتح `QFileDialog` لحفظ الملف مكان يختاره المستخدم (تحسين عن التحميل التلقائي في المتصفح).

### 3.4 شاشة تفاصيل الموظف (Employee Detail View)
- زر "رجوع" أعلى الشاشة — لو فيه تعديلات معلَّقة، يظهر Dialog تأكيد (حفظ وانتقال / إلغاء / تجاهل وانتقال) بنفس المنطق الحالي بالظبط.
- **جدول أيام الشهر:** كل يوم فيه: الحالة، البصمات، الساعات، وأزرار تعديل سريع.
- **لوحة المراجعة (Review Panel):** قائمة البصمات المحتاجة مراجعة (بصمة واحدة بس، ناقصة حضور/انصراف) — كل عنصر بـ `QLineEdit` لإدخال الوقت الناقص + Dropdown لتحديد الدور (حضور/انصراف).
- **محرر اليوم (Day Editor):** `QDialog` منبثق لاستبدال يوم كامل (تغيير الحالة أو الأزواج يدويًا) — بديل `show_day_editor` الحالي.
- **Bulk Smart Apply:** زر يفتح `QDialog` بمعاينة (Preview Table) لكل التصحيحات المقترحة تلقائيًا (مطبَّقة / متجاهَلة) قبل التأكيد — بنفس منطق `bulk_smart_apply_all`.

### 3.5 شاشة الإعدادات
نافذة/تبويب منفصل لإعدادات الشهر الحالي:
- Slider حد Redistribute (cutoff hour).
- Slider حد Saturate.
- Checkbox + Slider لنظام التسامح مع المغادرة المبكرة.
- تُحفظ تلقائيًا في `file_settings` لكل ملف شهر (بدون زر حفظ منفصل — Auto-save عند التغيير، بنفس فلسفة `auto_save_file_state` الحالية).

## 4. الفروقات الجوهرية عن سلوك Streamlit (نقاط تحسين UX)

| المشكلة في Streamlit | الحل في Native |
|---|---|
| كل ضغطة زرار = rerun لكل الصفحة | تحديث الـ Widget المتأثر فقط عبر Signals |
| لا يوجد مؤشر تحميل حقيقي أثناء تحليل ملف كبير | `QProgressDialog` مع نسبة تقدّم فعلية |
| التمرير (Scroll) بيرجع لأعلى الصفحة بعد كل rerun (كان محتاج CSS خاص لمنعه) | غير موجود أصلًا — Qt لا يعيد رسم الشاشة كاملة |
| الشريط العائم (Floating Apply Bar) قد يتداخل مع عناصر تانية | شريط حالة ثابت (`QStatusBar`) في مكانه دائمًا |
| تحميل الملفات عبر رابط Download في المتصفح | `QFileDialog.getSaveFileName` — المستخدم يختار المكان مباشرة |
| حقن JS في iframe الأب لزرار Sidebar | غير مطلوب أصلًا — `QDockWidget` قابل للطي أصلًا في Qt |

## 5. الأولوية في التصميم

1. **الدقة قبل الجمال:** كل رقم في الشاشة الجديدة لازم يطابق نفس الرقم في نسخة Streamlit لنفس الملف المُدخَل.
2. **الاستجابة الفورية:** أي إجراء (بحث، فلترة، فتح تفاصيل موظف) يجب أن يظهر أثره في أقل من 100ms على الأجهزة المتوسطة.
3. **الحفاظ على "الإحساس" الحالي للمستخدم** (الألوان، الأيقونات، ترتيب الأقسام) — عشان العميل الحالي (اللي اتعوّد على الشكل ده) ميحسّش إنه بيستخدم برنامج مختلف تمامًا، غير إنه بقى أسرع وأثبت.
